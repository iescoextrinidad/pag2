Módulos MongoDB v4.4

Fundamentos, Arquitectura y Operaciones CRUD

Módulo 1: Fundamentos y Arquitectura

1.1 SQL vs. NoSQL (El cambio de chip)

En MongoDB 4, la clave es la flexibilidad. Mientras que SQL te obliga a definir una estructura rígida, aquí usamos colecciones.

  • SQL: Base de Datos > Tabla > Fila > Columna
  • MongoDB: Base de Datos > Colección > Documento > Campo/Valor

1.2 Documentos: JSON y BSON

Aunque escribimos en JSON, MongoDB 4 guarda la información en BSON (Binary JSON) para soportar datos más complejos como fechas o binarios de forma eficiente.

El campo _id: Es obligatorio y único. Si no lo pones, MongoDB genera un ObjectId automáticamente por ti.

Módulo 2: Operaciones CRUD

2.1 Create (Insertar Datos)

// Insertar un solo documento
db.inventario.insertOne({ item: "tester", cantidad: 5, estado: "disponible" });

// Insertar varios a la vez
db.inventario.insertMany([{ item: "cable", cant: 10 }, { item: "led", cant: 100 }]);

2.2 Read (Consultas y Filtros)

  • Simple: db.usuarios.find({ ciudad: "Tucumán" })
  • Proyección: db.usuarios.find({}, { nombre: 1, _id: 0 }) (Muestra nombre, oculta ID)
  • Formato: .pretty() (Hace que el resultado sea legible en consola)

2.3 Update (Actualizar)

// Cambiar un valor específico con $set
db.alumnos.updateOne({ _id: 1 }, { $set: { estado: "asistente" } });

// Incrementar un contador con $inc
db.stock.updateOne({ item: "LED" }, { $inc: { cantidad: 10 } });

2.4 Delete (Eliminar)

Usa deleteOne() para el primero que encuentre o deleteMany() para todos los que coincidan con el filtro.

Ejemplo Práctico: Laboratorio

Crea una base de datos llamada Laboratorio y sigue estos pasos en la consola:

  1. Inserta un tester y un osciloscopio.
  2. Consulta solo los equipos que están operativos.
  3. Actualiza la ubicación de un equipo usando su _id.
  4. Borra un componente marcado como "desechado".
